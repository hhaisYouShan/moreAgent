# V2.0 Dashboard / Web View PRD

## 目标

为 `moreagent` 设计一个本地 Dashboard / Web View，用于查看 runs、workflow report、gate、test、review、repair、merge readiness 和 JSON/debug 信息。

V2.0 只做设计，不实现代码。

## 背景

当前能力已经具备两层结构化数据基础：

- V1.8：`status --json`、`inspect --workflow --json`
- V1.9：`report --run <id> --json`

这意味着 Dashboard 不需要直接读 `sessions.json` 或扫描 artifact 目录，可以优先基于现有 CLI JSON 输出构建视图层。

## 用户问题

当前 CLI 适合诊断和验收，但有几个明显摩擦：

1. run 列表、run 详情、workflow report 分散在不同命令。
2. gate / test / review / repair / merge readiness 需要人工拼装。
3. full workflow run 的阶段状态不容易快速扫读。
4. repair session 很多时，文本输出阅读成本高。
5. JSON 虽然机器可读，但人工不方便直接浏览。

## 产品目标

Dashboard 解决以下场景：

1. 快速查看最近 runs，识别 completed / failed / running。
2. 打开某个 run 后，立即看到最终结论：
   - 是否可 resume
   - 是否可 merge
   - 卡在哪个 phase
   - gate 是否通过
   - test / review 最终状态
   - repair 发生了几轮
3. 在一个页面里同时看到：
   - run 基本信息
   - workflow report
   - sessions
   - repair 轨迹
   - merge readiness
   - 原始 JSON/debug view

## 非目标

V2.0 不做：

1. 登录
2. 远程服务
3. 多用户协作
4. 自动刷新轮询
5. WebSocket
6. 复杂图表
7. 在线编辑
8. 直接触发 `start / merge / clean`
9. 脱离 CLI 的独立后端

## 核心用户

1. 本地开发者
2. CLI 验收者
3. 需要快速判断 run 状态的人工 reviewer
4. 后续可能接入 CI artifact 浏览的人

## MVP 设计原则

1. 先读，不写
2. 先静态，不服务化
3. 先单机，不远程
4. 先可验收，不追求实时
5. 先复用现有 JSON，不新增第二套数据协议

## 页面结构

## 1. Dashboard 首页

页面目标：查看最近 runs，并进入某个 run 的详情。

布局建议：

- 左侧：Run 列表
- 右侧：默认显示最新 run 的摘要卡片

Run 列表字段：

- run id
- task
- status
- profile
- createdAt
- hasRepair
- canResume
- canMerge
- overallStatus
- recommendation

交互：

- 点击某个 run，进入 run 详情视图
- 支持筛选：
  - all
  - failed
  - running
  - completed
  - repair
  - full workflow

MVP 可以只做前端本地筛选，不需要服务端查询。

## 2. Run 详情页

页面目标：把 `status --run`、`report --run`、`inspect --workflow` 三类信息合并展示。

推荐结构：

### A. Header Summary

展示：

- run id
- task
- status
- profile
- createdAt
- overallStatus
- recommendation
- canResume
- canMerge

### B. Workflow Report 区块

展示：

- completedPhases
- failedPhase
- totalPhases
- completedCount

MVP 视图形式：

- 一条 phase timeline
- 已完成 phase 用绿色
- failed phase 用红色
- 未开始 phase 用灰色

不需要复杂图表，用普通块状标签即可。

### C. Gate / Test / Review 状态区块

展示四个核心判断：

- PRD Gate
- Tech Gate
- Test
- Review

状态值：

- `APPROVED`
- `CHANGES_REQUESTED`
- `PASS`
- `FAIL`
- `unknown`

MVP 直接用状态卡片，不做图表。

### D. Repair Sessions 区块

展示：

- hasRepair
- repairCount
- repairRounds
- repair session 列表
- lastFailed

列表项字段：

- agentName
- status
- startedAt
- completedAt
- durationSeconds
- error

目标：让用户快速知道：

- repair 是 tester 触发还是 reviewer 触发
- repair 做了几轮
- 最后失败在谁

### E. Merge Readiness 区块

展示：

- hasWorktree
- worktree path
- branch
- dirty
- mainClean
- canMerge
- blockedReason
- mainDirtyFiles

MVP 必须能回答：

1. 这个 run 现在能不能 merge
2. 不能 merge 是因为 run 未完成、worktree 丢失、还是主项目不干净

### F. Sessions 区块

展示 run.sessions 列表：

- agentName
- status
- durationSeconds
- artifactDir
- worktreePath
- runtimeSessionId
- error

MVP 支持按 agentName 排序，repair session 不折叠。

### G. JSON / Debug 区块

展示原始 JSON：

- `status --run <id> --json`
- `report --run <id> --json`
- `inspect --run <id> --workflow --json`

MVP 用 tabs 切换：

1. status JSON
2. report JSON
3. workflow JSON

## 数据来源

Dashboard 只依赖以下三条命令：

1. `moreagent status --json`
2. `moreagent report --run <id> --json`
3. `moreagent inspect --run <id> --workflow --json`

推荐数据流：

### 首页

- 先调用 `moreagent status --json`
- 得到最近 runs 列表

### 详情页

并行拉取：

- `moreagent report --run <id> --json`
- `moreagent inspect --run <id> --workflow --json`
- 如需要 session 级细节，再补 `moreagent status --run <id> --json`

说明：

- `report` 负责最终判断和 recommendation
- `inspect --workflow` 负责 workflow / gate / repair 结构
- `status --run` 负责 session 级详情和 worktree/runtime session

## 关键产品决策

## 决策 1：V2.0 MVP 不做实时 Dashboard Server

原因：

1. 当前目标是本地验收和浏览，不是长驻服务。
2. 已有 JSON CLI 足够作为数据层。
3. 本地 dev server 会引入端口、生命周期和测试复杂度。
4. 用户约束明确：不做复杂刷新。

## 决策 2：V2.0 MVP 优先做静态 HTML 输出

建议输出：

- 单文件 HTML dashboard
- 通过 CLI 生成
- 浏览器本地打开

原因：

1. 不新增依赖更容易成立
2. 无需服务进程
3. 易于产出 artifact
4. 适合验收和归档

## 决策 3：命令层建议新增 `moreagent dashboard`

但其 MVP 行为应是：

- 生成 HTML
- 可选打开本地文件

而不是先做 server。

建议命令形态：

```bash
moreagent dashboard
moreagent dashboard --latest
moreagent dashboard --run <id>
moreagent dashboard --output .moreagent/dashboard/latest.html
```

默认行为建议：

- 如果不给 `--run`，显示最近 runs 列表 + 默认选中 latest run
- 输出一个 HTML 文件

## MVP 范围

V2.0 MVP 包含：

1. Dashboard HTML 页面
2. Run 列表
3. Run 详情
4. Workflow Report 视图
5. Gate / Test / Review 状态卡
6. Repair Sessions 列表
7. Merge Readiness 区块
8. JSON / Debug tabs

V2.0 MVP 不包含：

1. 自动刷新
2. 交互式调用 CLI 按钮
3. 历史趋势统计
4. 多项目聚合
5. 图表库
6. 搜索全文 artifact

## 成功标准

一个验收者在打开 Dashboard 后，应能在 30 秒内回答：

1. 最近 run 是成功还是失败
2. 如果失败，失败在什么阶段
3. gate / test / review 最终结果是什么
4. 是否发生 repair
5. repair 做了几次
6. 当前是否可 merge
7. 下一步更合理的是 `resume`、`merge` 还是人工检查
