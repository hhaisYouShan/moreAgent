# V1.8 JSON Output Technical Plan

## 架构

```
现状: status.ts / inspect.ts → console.log("*: value") 混合格式

目标: status.ts / inspect.ts → buildRunModel() → { text renderer | json renderer }
```

### 新增文件

| 文件 | 职责 |
|---|---|
| `src/commands/status.ts` | 提取 `buildRunSummary(run)` + `buildRunDetail(run)` 纯函数 |
| `src/commands/inspect.ts` | 提取 `buildWorkflowModel(run)` 纯函数 |
| `src/output/json.ts` | `printJson(data)`: `console.log(JSON.stringify(data, null, 2))` |

### 修改文件

| 文件 | 改动 |
|---|---|
| `src/cli.ts` | 传递 `--json` flag 到 status/inspect |
| `src/commands/status.ts` | 新增 `buildRunSummary()` / `buildRunDetail()`；`statusCommand` 路由到 `printJson` 或原有文本输出 |
| `src/commands/inspect.ts` | 新增 `buildWorkflowModel()`；`inspectCommand` 路由 |
| `test/regression.test.js` | 新增 JSON parse 测试 |

## 实现步骤

### Step 1: 提取纯数据模型

`status.ts` 中 `printRunSummary(run)` 内部计算了 `canResume`, `canMerge`, `gates`, `repair`, `worktree` 等值。将这些计算提取为一个纯函数 `buildRunSummary(run): RunSummary`。

```typescript
interface RunSummary {
  run: {
    id: string;
    task: string;
    status: string;
    createdAt: string;
    profile: string;
    completed: boolean;
    failed: boolean;
    canResume: boolean;
    resumeBlockedReason: string;
    // ...
  };
}
```

同理 `printLatestRun(run)` 提取为 `buildRunDetail(run): RunDetail`，包含 sessions 数组。

### Step 2: 渲染分派

```typescript
export function statusCommand(options: StatusOptions): void {
  // ... find run ...
  if (options.json) {
    const model = options.summary
      ? buildRunSummary(run)
      : buildRunDetail(run);
    printJson(model);
    return;
  }
  // 原有文本输出
}
```

### Step 3: 错误 JSON

```typescript
function printJson(data: unknown): void {
  process.stdout.write(JSON.stringify(data, null, 2) + '\n');
}

function printJsonError(code: string, message: string): void {
  printJson({ error: { code, message } });
  process.exit(1);
}
```

### Step 4: CLI 路由

```typescript
// cli.ts
case 'status':
  statusCommand({
    ...,
    json: args.includes('--json'),
  });
```

## 重构范围

| 函数 | 当前 | 改动 |
|---|---|---|
| `printRunSummary` | void, 直接打印 | → `buildRunSummary(): RunSummary` + 文本渲染 wrapper |
| `printLatestRun` | void, 直接打印 | → `buildRunDetail(): RunDetail` + 文本渲染 wrapper |
| `printWorkflowInfo` | void, 直接打印 | → `buildWorkflowModel(): WorkflowModel` + 文本渲染 wrapper |
| `printRunList` | 直接打印 | 暂不做 JSON（列表模式日常使用少） |

## 测试方案

```javascript
// test/v1.8-json.test.js
test('status --latest --json: valid JSON', () => {
  const r = runCli(['status', '--latest', '--json']);
  const data = JSON.parse(r.stdout);
  assert(data.run.id, 'missing run.id');
  assert(typeof data.run.canResume === 'boolean', 'canResume not boolean');
});

test('status --run x --summary --json: valid JSON with gates', () => {
  const r = runCli(['status', '--run', 'test-run', '--summary', '--json']);
  const data = JSON.parse(r.stdout);
  assert(data.run.gates, 'missing gates');
});

test('status --latest-full --json on nonexistent: error JSON', () => {
  const r = runCli(['status', '--latest-full', '--json']);
  assert(r.status !== 0, 'should exit non-zero');
  const data = JSON.parse(r.stdout || '{}');
  assert(data.error, 'missing error object');
});
```

## 风险与缓解

| 风险 | 缓解 |
|---|---|
| 文本渲染函数 200+ 行，重构成模型 + 渲染工作量大 | 只重构 `printRunSummary`（~80 行）和 `printLatestRun`（~30 行），`printRunList` 暂保持纯文本 |
| 模型字段变化导致调用方 break | JSON schema 首版确定后标注 stable，新增字段用可选 |
| JSON 输出与旧文本输出格式不一致（如 duration 文本 vs 数值） | JSON 用原始数值（秒），文本用格式化字符串，各自独立 |
| 错误场景 JSON 不完整 | 用 `printJsonError()` 统一错误输出 |
