# V1.8 JSON Output Technical Plan

## 架构

```
现状: status.ts / inspect.ts → console.log("*: value") 混合格式

目标: status.ts / inspect.ts → buildRunModel() → { text renderer | json renderer }
```

### 新增文件

| 文件 | 职责 |
|---|---|
| `src/commands/status.ts` | 提取 `buildRunSummary(run)` + `buildRunDetail(run)` + `buildRunListItem(run)` 纯函数 |
| `src/commands/inspect.ts` | 提取 `buildWorkflowModel(run)` + `buildRunOverview(run)` 纯函数 |
| `src/output/json.ts` | `printJson(data)`: `process.stdout.write(JSON.stringify(data, null, 2) + '\n')`; `printJsonError(code, message)`: 输出 `{ error: { code, message } }` 并 `process.exit(1)` |

### 修改文件

| 文件 | 改动 |
|---|---|
| `src/cli.ts` | 传递 `--json` flag 到 status/inspect；顶层 catch 检测 `--json` 输出 error JSON |
| `src/commands/status.ts` | 新增 `buildRunSummary()` / `buildRunDetail()` / `buildRunListItem()`；命令内路由到 `printJson` 或原有文本输出 |
| `src/commands/inspect.ts` | 新增 `buildWorkflowModel()` / `buildRunOverview()`；命令内路由 |
| `test/regression.test.js` | 新增 JSON parse 测试 |

## 错误处理策略

**采用方案 B：cli.ts 统一 catch。**

```
statusCommand/inspectCommand 内部:
  - 业务错误（No runs / Not found）: 调用 printJsonError(code, message)
  - 其他错误: throw，交由 cli.ts 处理

cli.ts catch:
  if (args.includes('--json')) {
    printJsonError('INTERNAL_ERROR', err.message);
  } else {
    console.error(...); // 当前文本行为
    process.exit(1);
  }
```

这样 status/inspect 不需要每个分支都 try/catch JSON 错误处理。

## 实现步骤

### Step 1: 创建 `src/output/json.ts`

```typescript
export function printJson(data: unknown): void {
  process.stdout.write(JSON.stringify(data, null, 2) + '\n');
}

export function printJsonError(code: string, message: string): never {
  printJson({ error: { code, message } });
  process.exit(1);
}

export function isJsonMode(args: string[]): boolean {
  return args.includes('--json');
}
```

### Step 4: 业务错误 JSON

```typescript
if (!run) {
  if (options.json) {
    printJsonError('RUN_NOT_FOUND', `Run not found: ${options.run}`);
  }
  console.log(`Run not found: ${options.run}`);
  return;
}
```

### Step 5: cli.ts 顶层 catch

```typescript
try {
  // ... switch (command) ...
} catch (err: any) {

    printJsonError('INTERNAL_ERROR', err.message);
  }
  console.error(`Error: ${err.message}`);
  process.exit(1);
}
```

### Step 6: inspect.ts 同理

`buildWorkflowModel(run)` 和 `buildRunOverview(run)` 提取。

## 重构范围

| 函数 | 当前 | 改动 |
|---|---|---|
| `printRunSummary` | void, 直接打印 | → `buildRunSummary(): RunSummary` + 文本渲染 wrapper |
| `printLatestRun` | void, 直接打印 | → `buildRunDetail(): RunDetail` + 文本渲染 wrapper |
| `printRunList` | void, 直接打印 | → `buildRunListItem(): RunListItem` + 文本/JSON 双渲染 |
| `printWorkflowInfo` | void, 直接打印 | → `buildWorkflowModel(): WorkflowModel` + 文本渲染 wrapper |
| `printRunOverview` | void, 直接打印 | → `buildRunOverview()` + 文本渲染 wrapper |
| `cli.ts catch` | `console.error` | → 检测 `--json` 输出 error JSON |

## Duration 字段

| 输出模式 | 格式 | 字段名 |
|---|---|---|
| 文本 | `"32s"` / `"N/A"` | 原样 |
| JSON | `32` / `null` | `durationSeconds` |

JSON 用纯数值，方便机器计算和排序。

// test/v1.8-json.test.js (追加到现有 regression.test.js 或新文件)

test('status --json (list): valid JSON array', () => {
  const r = runCli(['status', '--json']);
  const data = JSON.parse(r.stdout);
  assert(Array.isArray(data.runs), 'runs should be array');

test('status --latest --json: valid JSON with run.id', () => {
  const r = runCli(['status', '--latest', '--json']);
  const data = JSON.parse(r.stdout);
  assert(typeof data.run.id === 'string', 'missing run.id');
  assert(typeof data.run.canResume === 'boolean', 'canResume not boolean');

test('status --run nonexistent --json: error JSON + exit 1', () => {
  const r = runCli(['status', '--run', 'does-not-exist', '--json']);
  assert(r.status !== 0, 'should exit non-zero');
  const data = JSON.parse(r.stdout);
  assert(data.error, 'missing error object');
  assert(data.error.code === 'RUN_NOT_FOUND', `expected RUN_NOT_FOUND, got ${data.error.code}`);
});

test('inspect --run nonexistent --json: error JSON + exit 1', () => {
  const r = runCli(['inspect', '--run', 'does-not-exist', '--json']);
  assert(r.status !== 0, 'should exit non-zero');
  const data = JSON.parse(r.stdout);
  assert(data.error, 'missing error object');
});

test('status --latest --json: durationSeconds is number', () => {
  const r = runCli(['status', '--latest', '--json']);
  const data = JSON.parse(r.stdout);
  for (const s of (data.run.sessions || [])) {
    if (s.durationSeconds !== null) {
      assert(typeof s.durationSeconds === 'number', 'durationSeconds should be number');
    }
  }
});

test('non-json mode: errors still go to stderr as text', () => {
  const r = runCli(['status', '--run', 'does-not-exist']);
  assert(r.status !== 0 || r.stderr.length > 0 || r.stdout.includes('not found'),
    'non-json error should be text');
});

## 风险与缓解

| 风险 | 缓解 |
|---|---|
| 文本渲染函数 200+ 行，重构成模型 + 渲染工作量大 | 只重构 `printRunSummary`（~80 行）、`printLatestRun`（~30 行）、`printRunList`（~20 行），不碰内部 helper |
| 模型字段变化导致调用方 break | JSON schema 首版确定后标注 stable，新增字段用可选 |
| JSON 错误码不一致 | `printJsonError` 统一入口，定义错误码枚举 |
| 非 JSON 模式错误行为变化 | `cli.ts` catch 中区分 `--json` 检测，非 JSON 模式完全保持原行为 |
