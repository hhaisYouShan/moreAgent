# V1.8 JSON Output PRD

## 目标

为 `moreagent status` 和 `moreagent inspect` 新增 `--json` 输出模式，方便 dashboard、CI、自动化验收和多 agent 调度消费结构化数据。

## 范围

| 命令 | 模式 |
|---|---|
| `moreagent status --latest --json` | JSON 单 run |
| `moreagent status --latest-repair --json` | JSON 单 run |
| `moreagent status --latest-full --json` | JSON 单 run |
| `moreagent status --run <id> --json` | JSON 单 run |
| `moreagent status --run <id> --summary --json` | JSON summary（紧凑） |
| `moreagent inspect --run <id> --json` | JSON 单 run |
| `moreagent inspect --run <id> --workflow --json` | JSON workflow 详情 |

## 兼容性

- 默认文本输出不变
- `--json` 下 stdout 只输出 JSON（一行或多行 format），stderr 留给错误
- `process.exit(0)` 正常，`process.exit(1)` 错误（stdout 可能为空，stderr 输出错误对象）

## JSON Schema

```json
{
  "run": {
    "id": "run-2026-06-30T02-40-35-8178c8",
    "task": "...",
    "status": "completed",
    "createdAt": "2026-06-30T02:40:35.310Z",
    "profile": "full",
    "completed": true,
    "failed": false,
    "canResume": false,
    "resumeBlockedReason": "run is completed",
    "canMerge": true,
    "mergeBlockedReason": "",
    "hasRepair": false,
    "repairSessions": { "total": 0, "failureAnalysis": 0, "repairs": 0, "retries": 0, "revisions": 0 },
    "hasGate": true,
    "gates": { "prdGate": "APPROVED", "techGate": "APPROVED", "test": "PASS", "review": "APPROVED" },

  }

```

### Run Detail（status --latest 级）

Summary 基础上嵌套 sessions 数组：

```json
{
  "run": {
    "...": "...",
    "completedPhases": ["brain", "prd", "..."],
    "failedPhase": null,
    "sessions": [
      {
        "agentName": "brain",
        "status": "completed",
        "duration": "32s",
        "startedAt": "...",
        "completedAt": "...",
        "artifactDir": "...",
        "runtimeSessionId": "ses_xxx",
        "worktreePath": "...",
        "error": null
      }
    ]
  }
}
```

### Workflow Detail（inspect --workflow 级）

```json
{
  "run": { "...": "..." },
  "workflow": {
    "profile": "full",
    "currentPhase": null,
    "completedPhases": ["brain", "prd", "..."],
    "failedPhase": null,
    "gates": { "...": "..." },
    "repairSessions": [
      { "agentName": "repair-1-implementer", "status": "completed" }
    ],
    "lastFailed": null,
    "suggestedCommands": ["moreagent inspect --run ...", "..."]
  }
}
```

### 错误 JSON

```json
{
  "error": {
    "message": "No runs found.",
    "code": "NO_RUNS"
  }
}
```

## 持久化模型

当前 `status.ts` 和 `inspect.ts` 直接在 console.log 中渲染格式化字符串。V1.8 应内部提取一个纯函数 `buildRunSummary(run): RunSummary`，返回纯数据对象，然后：
- 文本渲染器：用当前逻辑格式化输出
- JSON 渲染器：`JSON.stringify(summary, null, 2)`

避免把展示字符串反解析成 JSON。

## 验收标准

1. 所有现有文本输出不变
2. `--json` 在 stdout 输出合法 JSON
3. `node -e "require('./output.json')"` 可 parse
4. schema 字段名稳定（首版即确定）
5. MVP run 和 full workflow run 均能正常输出 JSON
6. 错误场景 stderr 有错误信息，stdout 有合法 JSON 错误对象

## 不做

- 不做 dashboard
- 不做 Web UI
- 不做二进制输出（protobuf/msgpack）
- 不做流式 NDJSON
- 不改变现有 API
