# V1.9 Workflow Report PRD

## 目标

为 `moreagent` 新增标准化 `report` 命令，输出一次 run 的完整验收报告，供 CLI 人工阅读、Dashboard 可视化、CI 集成和自动合并判断使用。

基于 V1.8 JSON 输出能力，`report` 是 `status --summary --json` 的上层封装：复用同一数据模型，增加 recommendation 决策字段。

## CLI 行为

| 命令 | 输出 |
|---|---|
| `moreagent report --latest` | 文本报告，latest run |
| `moreagent report --run <id>` | 文本报告，指定 run |
| `moreagent report --latest --json` | JSON 报告，latest run |
| `moreagent report --run <id> --json` | JSON 报告，指定 run |

- 默认输出文本报告（适合人读）。
- `--json` 输出纯 JSON（适合机器消费）。
- 错误场景输出 `{ error: { code, message } }`，exit 1。
- 复用 V1.8 错误码：`NO_RUNS`, `RUN_NOT_FOUND`, `INTERNAL_ERROR`。

## JSON Schema

```json
{
  "report": {
    "run": {
      "id": "run-2026-06-30T02-40-35-8178c8",
      "task": "创建 README.md...",
      "status": "completed",
      "createdAt": "2026-06-30T02:40:35.310Z",
      "profile": "full"
    },
    "workflow": {
      "completedPhases": ["brain", "prd", "prd-review", "prd-gate", "tech-plan", "tech-gate", "implementation", "test", "review"],
      "failedPhase": null,
      "totalPhases": 9,
      "completedCount": 9
    },
    "gates": {
      "prdGate": "APPROVED",
      "techGate": "APPROVED"
    },
    "quality": {
      "test": "PASS",
      "review": "APPROVED",
      "hasRepair": false,
      "repairCount": 0,
      "repairRounds": 0
    },
    "worktree": {
      "hasWorktree": true,
      "exists": true,
      "path": "/path/to/worktree",
      "branch": "agent/run-xxx",
      "dirty": false
    },
    "merge": {
      "canMerge": true,
      "blockedReason": "",
      "mainClean": false,
      "mainDirtyFiles": [".moreagent/sessions.json"]
    },
    "decision": {
      "canResume": false,
      "overallStatus": "PASSED",
      "recommendation": "MERGE_READY"
    }
  }
}
```

### 字段说明

| 路径 | 类型 | 说明 |
|---|---|---|
| `report.run` | object | 基本信息（id, task, status, createdAt, profile） |
| `report.workflow` | object | 阶段进度（completedPhases, failedPhase, totalPhases, completedCount） |
| `report.gates` | object | Gate 决策（prdGate, techGate: APPROVED / CHANGES_REQUESTED / unknown） |
| `report.quality` | object | 质量指标（test: PASS/FAIL/unknown, review: APPROVED/CHANGES_REQUESTED/unknown, hasRepair, repairCount） |
| `report.worktree` | object | Worktree 信息（hasWorktree, exists, path, branch, dirty） |
| `report.merge` | object | 合并就绪（canMerge, blockedReason, mainClean, mainDirtyFiles） |
| `report.decision` | object | 最终决策 |

### overallStatus 判定顺序

按以下优先级依次判断，命中即返回：

| 优先级 | 值 | 条件 |
|---|---|---|
| 1 | `RUNNING` | `run.status === "running"` |
| 2 | `FAILED` | `run.status === "failed"` 或 `test === "FAIL"` 或 `review === "CHANGES_REQUESTED"` 或任一 gate === `"CHANGES_REQUESTED"` |
| 3 | `PARTIAL` | `run.status === "completed"` 且 test/review/gate 存在 `unknown`（缺少判定标记） |
| 4 | `PASSED` | `run.status === "completed"`，test === `"PASS"`，review === `"APPROVED"`，所有必要 gate === `"APPROVED"` |
| 5 | `UNKNOWN` | 其他（如 pending） |

注意：`unknown` 不直接判 `FAILED`。缺失判定标记只降级为 `PARTIAL`。

### recommendation 定义

| 值 | 条件 |
|---|---|
| `MERGE_READY` | overallStatus === PASSED && canMerge && worktree.exists && mainClean |
| `BLOCKED` | overallStatus === PASSED 但 canMerge === false 或 worktree missing 或 mainClean === false |
| `NEEDS_REPAIR` | overallStatus === FAILED && canResume === true |
| `NEEDS_REVIEW` | overallStatus === PARTIAL（需人工检查缺失判定标记） |
| `RUNNING` | overallStatus === RUNNING |
| `UNKNOWN` | 其他 |

## 文本报告示例

```
Workflow Report
Run ID:       run-2026-06-30T02-40-35-8178c8
Task:         创建 README.md，内容只包含项目名称 bossResume 和 Usage 示例
Status:       completed
Profile:      full
Created At:   2026-06-30T02:40:35.310Z

Workflow Progress: 9/9 phases completed
  brain → prd → prd-review → prd-gate → tech-plan → tech-gate
  → implementation → test → review

Gate Decisions:
  PRD Gate:     APPROVED
  Tech Gate:    APPROVED

Quality:
  Test:         PASS
  Review:       APPROVED
  Repairs:      0

Merge Readiness:
  Can Merge:    yes
  Worktree:     /path (branch: agent/run-xxx)
  Main Clean:   no (.moreagent/sessions.json)

Decision:
  Overall:      PASSED
  Recommend:    MERGE_READY
```

## 实现策略

**`report` 命令复用 V1.8 的 `buildRunSummary()` 数据模型**，在其基础上追加：

- `merge.mainClean` / `merge.mainDirtyFiles`：通过 `git status --porcelain` 检查
- `workflow.totalPhases` / `workflow.completedCount`：从 completedPhases 计算
- `decision.overallStatus` / `decision.recommendation`：纯函数，基于已有字段计算

不需引入新的数据读取逻辑。

## 测试方案

| 场景 | overallStatus | recommendation | 说明 |
|---|---|---|---|
| completed + PASS/APPROVED + main clean + canMerge | PASSED | MERGE_READY | 最优路径 |
| completed + PASS/APPROVED + main dirty | PASSED | BLOCKED | main 不 clean |
| completed + PASS/APPROVED + worktree missing | PASSED | BLOCKED | worktree 不存在 |
| completed + test unknown + review APPROVED | PARTIAL | NEEDS_REVIEW | 缺少 tester 判定标记 |
| completed + review unknown | PARTIAL | NEEDS_REVIEW | 缺少 reviewer 判定标记 |
| completed + test FAIL | FAILED | NEEDS_REPAIR | tester 判定失败 |
| completed + review CHANGES_REQUESTED | FAILED | NEEDS_REPAIR | reviewer 判定失败 |
| running | RUNNING | RUNNING | 正在执行 |
| failed + canResume | FAILED | NEEDS_REPAIR | 可恢复执行 |
| no runs | — | — | error JSON: NO_RUNS |
| run not found | — | — | error JSON: RUN_NOT_FOUND |
| JSON parse | — | — | `JSON.parse(stdout)` 合法 |
| 文本输出 | — | — | 包含 "Workflow Report" / "Decision" / "Recommend" |
