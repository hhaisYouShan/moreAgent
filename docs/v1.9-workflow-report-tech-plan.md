# V1.9 Workflow Report Technical Plan

## 架构

```
report.ts:
  buildReport(run) → { report: {...} }
    ├── 直接读 run.workflow?.completedPhases / failedPhase
    ├── 调用 status.ts 内部 helpers（getGateSummary, isRepairRun, checkCanResume, checkCanMerge, getWorktreeInfo）
    ├── git status --porcelain 获取 main clean 状态
    └── computeDecision() 计算 overallStatus / recommendation
```

`report.ts` 不依赖 V1.8 的 `buildRunSummary()`，直接从 `Run` 读取原始数据。

### Step 1: 定义类型

```typescript
// src/output/report.ts
export interface ReportModel {
  report: {
    run: { id, task, status, createdAt, profile };
    workflow: { completedPhases, failedPhase, totalPhases, completedCount };
    gates: { prdGate, techGate };
    quality: { test, review, hasRepair, repairCount, repairRounds };
    worktree: { hasWorktree, exists, path, branch, dirty };
    merge: { canMerge, blockedReason, mainClean, mainDirtyFiles };
    decision: { canResume, overallStatus, recommendation };
  };
}
```

### Step 2: buildReport(run)

```typescript
function buildReport(run: Run): ReportModel {
  const gates = getGateSummary(run);
  const worktree = getWorktreeInfo(run);
  const canMerge = checkCanMerge(run);
  const canResume = checkCanResume(run);
  const mainStatus = execSync('git status --porcelain', { encoding: 'utf-8' }).trim();
  const mainClean = mainStatus.length === 0;
  const mainDirtyFiles = mainClean ? [] : mainStatus.split('\n').filter(Boolean);

  return {
    report: {
      run: {
        id: run.id, task: run.task, status: run.status,
        createdAt: run.createdAt,
        profile: run.workflow?.profile ?? (isFullWorkflowRun(run) ? 'full' : 'mvp'),
      },
      workflow: {
        completedPhases: run.workflow?.completedPhases ?? [],
        failedPhase: run.workflow?.failedPhase ?? null,
        totalPhases: FULL_WORKFLOW_PHASES.length,
        completedCount: (run.workflow?.completedPhases ?? []).length,
      },
      gates,
      quality: {
        test: gates.test,
        review: gates.review,
        hasRepair: isRepairRun(run),
        repairCount: countRepairSessions(run),
        repairRounds: countRepairRounds(run),
      },
      worktree,
      merge: { canMerge: canMerge.ok, blockedReason: canMerge.reason, mainClean, mainDirtyFiles },
      decision: computeDecision(run, gates, canMerge.ok, canResume.ok, worktree, mainClean),
    },
  };
}
```

所有数据直接从 `Run` 对象、`git status` 和内联 helper 读取，不经过 `buildRunSummary`。

### Step 3: computeDecision(run, gates, canMerge, canResume, worktree, mainClean)

```typescript
function computeDecision(run, gates, canMerge, canResume, worktree, mainClean) {
  const testOk = gates.test === 'PASS';
  const reviewOk = gates.review === 'APPROVED';
  const gatesOk = gates.prdGate !== 'CHANGES_REQUESTED' && gates.techGate !== 'CHANGES_REQUESTED';
  const anyUnknown = gates.test === 'unknown' || gates.review === 'unknown'
    || gates.prdGate === 'unknown' || gates.techGate === 'unknown';

  // Priority: RUNNING → FAILED → PARTIAL → PASSED → UNKNOWN
  let overallStatus;
  if (run.status === 'running') {
    overallStatus = 'RUNNING';
  } else if (run.status === 'failed' || !testOk || !reviewOk || !gatesOk) {
    overallStatus = 'FAILED';
  } else if (run.status === 'completed' && anyUnknown) {
    overallStatus = 'PARTIAL';
  } else if (run.status === 'completed' && testOk && reviewOk && gatesOk) {
    overallStatus = 'PASSED';
  } else {
    overallStatus = 'UNKNOWN';
  }

  let recommendation;
  if (overallStatus === 'RUNNING') {
    recommendation = 'RUNNING';
  } else if (overallStatus === 'PASSED' && canMerge && worktree.exists && mainClean) {
    recommendation = 'MERGE_READY';
  } else if (overallStatus === 'PASSED' && (!canMerge || !worktree.exists || !mainClean)) {
    recommendation = 'BLOCKED';
  } else if (overallStatus === 'FAILED' && canResume) {
    recommendation = 'NEEDS_REPAIR';
  } else if (overallStatus === 'PARTIAL') {
    recommendation = 'NEEDS_REVIEW';
  } else {
    recommendation = 'UNKNOWN';
  }

  return { canResume, overallStatus, recommendation };
}
```

## 函数复用策略

**不 export status.ts 内部函数。** `report.ts` 内联所有需要的 helper：

| 函数 | 方案 |
|---|---|
| `getGateSummary` | 内联简化版（扫描 sessions 读取 artifact 文件） |
| `isRepairRun` | 内联（3 行逻辑） |
| `checkCanResume` | 内联（5 行逻辑） |
| `checkCanMerge` | 内联（5 行逻辑） |
| `getWorktreeInfo` | 内联（git 命令 + fs 检查） |
| `getMainGitStatus` | 内联（`git status --porcelain`） |
| `isFullWorkflowRun` | 内联（检查 sessions 名称） |

如果未来多个模块需要复用同一 helper，再提取到 `src/report/model.ts`。

test('report on completed + PASS/APPROVED + clean → MERGE_READY', () => {

  const data = JSON.parse(r.stdout);
  assert(data.report.decision.overallStatus === 'RUNNING');
