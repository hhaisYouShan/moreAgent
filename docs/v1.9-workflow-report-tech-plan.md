### Step 2: buildReport(run)

```typescript
function buildReport(run: Run): ReportModel {
  const summary = buildRunSummary(run); // from V1.8 status.ts
  const mainStatus = getMainGitStatus();
  const decision = computeDecision(summary, run);

  return {
    report: {
      run: { id: run.id, task: run.task, status: run.status, createdAt: run.createdAt, profile: summary.profile },
      workflow: {
        completedPhases: summary.completedPhases || [],
        failedPhase: summary.failedPhase || null,
        totalPhases: FULL_PHASES.length,
        completedCount: (summary.completedPhases || []).length,
      },
      gates: summary.gates,
      quality: {
        test: summary.gates.test,
        review: summary.gates.review,
        hasRepair: summary.hasRepair,
        repairCount: summary.repairSessions?.total || 0,
        repairRounds: Math.ceil((summary.repairSessions?.total || 0) / 3), // rough estimate
      },
      worktree: summary.worktree,
      merge: {
        canMerge: summary.canMerge,
        blockedReason: summary.mergeBlockedReason,
        mainClean: mainStatus.trim().length === 0,
        mainDirtyFiles: mainStatus.trim().split('\n').filter(Boolean),
      },
      decision,
    },
  };
}
```

### Step 3: computeDecision(summary, run)

```typescript
function computeDecision(summary, run): Decision {
  const testOk = summary.gates.test === 'PASS';
  const reviewOk = summary.gates.review === 'APPROVED';
  const gatesOk = summary.gates.prdGate !== 'CHANGES_REQUESTED'
    && summary.gates.techGate !== 'CHANGES_REQUESTED';
  const passed = run.status === 'completed' && testOk && reviewOk && gatesOk;
  const failed = run.status === 'failed' || !testOk || !reviewOk || !gatesOk;
  const partial = run.status === 'completed' && (summary.gates.test === 'unknown' || summary.gates.review === 'unknown');

  const overallStatus = passed ? 'PASSED' : failed ? 'FAILED' : run.status === 'running' ? 'RUNNING' : partial ? 'PARTIAL' : 'UNKNOWN';
  const recommendation = overallStatus === 'PASSED' && summary.canMerge ? 'MERGE_READY'
    : overallStatus === 'FAILED' && summary.canResume ? 'NEEDS_REPAIR'
    : overallStatus === 'PARTIAL' ? 'NEEDS_REVIEW'
    : overallStatus === 'PASSED' && !summary.canMerge ? 'BLOCKED'
    : overallStatus === 'RUNNING' ? 'RUNNING'
    : 'UNKNOWN';

  return { canResume: summary.canResume, overallStatus, recommendation };
}
```

### Step 6: 导出 buildRunSummary

当前 `buildRunSummary` 在 `status.ts` 中为内部函数。需要导出供 `report.ts` 使用：

```diff
-function buildRunSummary(run: Run)
+export function buildRunSummary(run: Run)
```

同理 `getMainGitStatus()` 需要从 `merge.ts` 提取或新增。

## 函数导出需求

| 函数 | 来源 | 改动 |
|---|---|---|
| `buildRunSummary()` | `status.ts` | 加 `export` |
| `getProfile()` | `status.ts` | 加 `export`（如果需要独立使用） |
| `checkCanResume()` | `status.ts` | 加 `export` |
| `checkCanMerge()` | `status.ts` | 加 `export` |
| `isRepairRun()` | `status.ts` | 加 `export` |
| `getMainGitStatus()` | `merge.ts` | 新增导出或提取到共享模块 |

**优先方案**：不修改 `status.ts` 的导出。`report.ts` 内部复制 `buildRunSummary` 的调用（通过 `status.ts` 的公开导出，或直接在 `report.ts` 中实现独立的 `buildReportData`）。

**最小侵入方案**：`report.ts` 中直接读取 run 数据并构建 report，复用 `status.ts` 中已有的公开 helper 函数（如 `isRepairRun`, `checkCanResume` 等），仅导出必要的函数。

具体选择：
- `isRepairRun`, `isFullWorkflowRun`, `checkCanResume`, `checkCanMerge`, `getWorktreeInfo`, `getGateSummary`, `getRepairSummary`, `getProfile`, `getNextActions` → 已为内部函数，需要 export
- 如果 export 影响太大，可在 `report.ts` 中内联一份简化版

**建议**：给 8 个函数的声明加 `export`，向下兼容，不影响现有调用。
