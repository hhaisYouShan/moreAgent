# Validation

This directory contains real-project validation packs for AI Software Company OS.

Validation begins only after Stage 6 system-level testing is approved. A validation pack may contain project profiles, adapters, scenarios, benchmark definitions, expected evidence, and validation findings.

Rules:

- Consumer business PRDs and business code remain in the consumer repository.
- Validation material cannot change Core contracts or state semantics.
- Project-specific findings must be classified as Core, Profile, Adapter, documentation, or consumer-project issues.
- BossResume is the Stage 7 first real-project validation target.
